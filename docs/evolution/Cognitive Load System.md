# Cognitive Load System

Pickla should become easier as it becomes more capable.

The Cognitive Load System is how Pickla measures that promise.

## Decision Count

Decision Count is the number of choices a person must make to move forward.

Good product work reduces unnecessary decisions.

Examples:

- The system chooses the right price.
- Members do not choose between products they already own.
- Staff see the next operational action instead of a dashboard of counts.

## Reading Count

Reading Count is the amount of text required to understand the screen.

Text is sometimes necessary.

But if a feature needs paragraphs to be understood, the model or hierarchy is probably wrong.

Good screens use fewer words because the structure carries meaning.

## Tap Count

Tap Count is the number of actions required to complete the real job.

It should be measured from the user's actual starting point.

Not:

"From the modal."

But:

"From the moment the customer arrives."

For staff, live operations should rarely require hunting.

## Thinking Count

Thinking Count is the number of mental translations required.

Examples of unnecessary thinking:

- Translating product keys into customer meaning
- Comparing several price cards
- Understanding whether membership applies
- Deciding whether a customer is allowed to check in
- Guessing whether a booking is paid

The resolver should do this work.

## Waiting Count

Waiting Count is the number of moments where the user does not know what is happening.

Some waiting is technical.

Unclear waiting is design debt.

A good waiting state says:

- What is happening
- Whether action is needed
- What happens next

## The Rule

Every feature should reduce at least one count over time.

If a feature increases Decision Count, Reading Count, Tap Count, Thinking Count, and Waiting Count, it is not mature enough.

## Review Questions

Before release, ask:

- What decision disappeared?
- What text disappeared?
- What tap disappeared?
- What thinking disappeared?
- What waiting became clearer?

If nothing disappeared, the work is not finished.
